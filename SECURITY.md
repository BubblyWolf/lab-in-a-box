# Security Policy

## Reporting a Vulnerability

Please **do not** open a public issue for security vulnerabilities.

Instead, report them privately through GitHub:

1. Go to the **[Security tab](https://github.com/BubblyWolf/lab-in-a-box/security)** of this repository.
2. Click **"Report a vulnerability"** (GitHub Private Vulnerability Reporting).
3. Describe the issue, affected versions, and reproduction steps.

You'll receive a response as soon as possible. Reports are handled confidentially, and you'll be credited (if you wish) once a fix is released.

## Scope

`lab-in-a-box` is a local learning platform intended to run on a developer's own machine via Kind. It is **not** designed or hardened for production or internet-facing use. Please keep that context in mind when reporting — issues that only apply to unsupported production deployments may be documented rather than fixed.

## Supported Versions

This is an actively developed learning project. Security fixes are applied to the `main` branch.

Thank you for helping keep the project and its users safe. 🙏
